# 全栈用户管理系统 - Claude Code 项目配置

## 项目概述

本项目是一个完整的全栈用户管理系统，采用 FastAPI 后端和 Next.js 前端。系统提供完整的用户管理功能，包括用户注册、登录、权限管理、角色分配、个人信息管理等功能。

### 核心目标
1. 提供一个现代化、可扩展的用户管理系统模板
2. 实现前后端分离的清晰架构
3. 确保高安全性和良好的用户体验
4. 提供完整的测试和部署方案

## 技术栈选择

### 后端 (FastAPI)
- **框架**: FastAPI (Python 3.11+)
- **数据库**: PostgreSQL 14+ (生产环境), SQLite (开发环境)
- **ORM**: SQLAlchemy 2.0 + Alembic 迁移
- **认证**: JWT + OAuth2 with Password Bearer
- **验证**: Pydantic V2
- **异步**: async/await with asyncpg

### 前端 (Next.js)
- **框架**: Next.js 14 (App Router)
- **语言**: TypeScript 5+
- **UI 库**: shadcn/ui + Tailwind CSS
- **状态管理**: Zustand
- **表单处理**: React Hook Form + Zod
- **HTTP 客户端**: Axios

### 基础设施
- **容器化**: Docker + Docker Compose
- **CI/CD**: GitHub Actions
- **监控**: OpenTelemetry + Prometheus
- **日志**: structlog + JSON 格式

## 开发原则

### 代码质量
1. **类型安全**: 全面使用 TypeScript 和 Python 类型提示
2. **代码规范**: 遵循 Prettier、ESLint、Black、isort 规范
3. **测试驱动**: 测试覆盖率 > 85%，包含单元测试、集成测试和 E2E 测试
4. **文档完整**: 所有公共 API 都有 OpenAPI 文档，重要函数都有类型和文档字符串

### 架构原则
1. **关注点分离**: 清晰的层级结构（路由 → 服务 → 仓储 → 模型）
2. **依赖注入**: 使用 FastAPI 的 Depends 实现松耦合
3. **单一职责**: 每个组件只做一件事，且做好
4. **开闭原则**: 对扩展开放，对修改关闭

### 安全原则
1. **最小权限**: 所有操作都需要明确的权限检查
2. **输入验证**: 所有用户输入都必须经过严格的验证
3. **安全默认值**: 默认拒绝，需要时明确允许
4. **防御性编程**: 假设所有外部输入都是恶意的

## Claude Code 行为约束

### 权限管理
1. **文件操作**: 可以创建、修改项目相关文件，但不能删除核心配置文件
2. **依赖管理**: 可以添加、更新依赖，但需要记录在 CHANGELOG 中
3. **数据库变更**: 所有数据库变更必须通过 Alembic 迁移进行
4. **敏感信息**: 绝不能硬编码密钥、密码等敏感信息

### 开发流程
1. **渐进式开发**: 按照提示链的顺序逐步实现功能
2. **代码审查**: 每个重要功能提交前都需要自我审查
3. **测试先行**: 实现功能前先写测试（如果适用）
4. **文档同步**: 代码变更时同步更新相关文档

### 错误处理
1. **优雅降级**: 系统部分失败时不应导致整体崩溃
2. **明确错误**: 错误信息应该对用户友好，对开发者详细
3. **日志记录**: 所有错误都应该被记录，包含足够上下文
4. **监控告警**: 关键错误应该触发监控告警
